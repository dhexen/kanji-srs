# 08 — Internacionalización (i18n)

## Idiomas soportados

| Código | Idioma | Nombre mostrado |
|--------|--------|-----------------|
| `es` | Español | 🇪🇸 Español |
| `ca` | Catalán | 🏴󠁥󠁳󠁣󠁴󠁿 Català |
| `en` | Inglés | 🇬🇧 English |
| `ja` | Japonés | 🇯🇵 日本語 |

## Uso

```typescript
import { t } from '@/lib/i18n'

// En un componente:
const lang = state.lang  // 'es' | 'ca' | 'en' | 'ja'
const texto = t(lang, 'review_correct')  // "¡Excelente! 🎉 Respuesta correcta."

// Con interpolación:
t(lang, 'vocab_tab_n').replace('{n}', String(count))
```

## Cambiar el idioma de la interfaz

El usuario puede cambiar el idioma en `/stats` → pestaña Configuración. El idioma se guarda en `user_settings.language` en Supabase y se sincroniza al cargar la sesión.

## Añadir nuevas traducciones

1. Añadir la clave en los 4 idiomas en `lib/i18n.ts`
2. La función `t(lang, key)` devuelve `''` si la clave no existe (sin error)
3. Usar `as Parameters<typeof t>[1]` para type-safety si TypeScript lo exige

```typescript
// Forma type-safe:
t(lang as Parameters<typeof t>[0], 'mi_nueva_clave' as Parameters<typeof t>[1])

// Forma corta (si el componente ya conoce el tipo):
tl('mi_nueva_clave')  // donde tl = (key: string) => t(lang, key as ...)
```
